# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:

- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:

- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Данные. Типы Данных

Интерфейс Продукта

```
interface IProduct {
	id: string;
	description: string;
	image: string;
	title: string;
	category: string;
	price: number;
}
```

Интерфейс информации покупателя

```
interface IUser {
	payment: string;
	address: string;
	email: string;
	phone: string;
	total: number;
	items: [];
}
```

Интерфейс Католог карточек

```
export interface IProductsData {
	items: IProduct[];
	preview: string | null;
	setProducts(items: IProduct[]): void;
	getProducts(): IProduct[];
	getProduct(id: string): IProduct;
}
```

Интерфейс Католога корзины

```
interface ICartData {
	products: IProduct[];
	total: number;
	getTotalPrice(): TCartTotal;
	getCartList(): IProduct[];
	addProduct: (product: IProduct) => void;
	deleteProduct: (id: string) => void;
	clearCart: () => void;
}
```

Данные общей стоимости товаров

```
export type TCartTotal = Pick<ICartData, "total">
```

Данные Формы с методом оплаты и адрессом

```
type TFormPayInfo = Pick<IFormInfo, 'payMethod' | 'address'>;
```

Данные Формы с методом email и телефоном

```
type TFormContactInfo = Pick<IFormInfo, 'email' | 'phone'>;
```

### Базовый код

#### Класс Api

Содержит в себе базовую логику отправки запросов. В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.
Методы:

- `get` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер
- `post` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.

#### Класс EventEmitter

Брокер событий позволяет отправлять события и подписываться на события, происходящие в системе. Класс используется в презентере для обработки событий и в слоях приложения для генерации событий.  
Основные методы, реализуемые классом описаны интерфейсом `IEvents`:

- `on` - подписка на событие
- `emit` - инициализация события
- `trigger` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие

### СЛОЙ МОДЕЛЬ

#### Класс ProductData

Класс отвечает за работу с продуктами.\
Конструктор класса принимает инстант брокера событий\
в полях класса хранятся следующие данные:

- \_products: IProduct[] - массив обьектов продуктов
- \_preview: string | null - id карточки продукта выбранной для просмотра в модальном окне
- events: IEvents - экземпляр класса `EventEmitter`для инициации событий при изменении данных.

Так же класс предоставляет набор методов для взаимодействия с этими данными.

- getProduct(id: string): IProduct - возвращает карточку по ее id
- а так-же сеттеры и геттеры для сохранения и получения данных из полей класса

#### Класс CartData

Класс отвечает за работу с корзиной.\
Конструктор класса принимает инстант брокера событий\
в полях класса хранятся следующие данные:

- \_products: IProduct[] - массив обьектов продуктов
- events: IEvents - экземпляр класса `EventEmitter`для инициации событий при изменении данных.

Так же класс предоставляет набор методов для взаимодействия с этими данными.

- getTotalPrice(): number - для получения общей суммы заказа
- addProduct: (product: IProduct): void - добавляет один продукт в начало массива
- deleteProduct: (productId: string, payload: Function | null = null) => void - удаляет продукт из массива. Если передан колбэк, то выполняет его после удаления, если нет, то вызывает событие изменения массива.
- clearCart: () => void; удаляет все продукты из корзины
- а так-же сеттеры и геттеры для сохранения и получения данных из полей класса

#### Класс UserData

Класс отвечает за работу с Данными покупателя.\
Конструктор класса принимает инстант брокера событий\
в полях класса хранятся следующие данные:

- payMethod: string - Способ оплаты
- address: string - адресс
- email: string - email покупателя
- phone: number - номер
- total: number - общая сумма заказа
- items: [] - массив с id выбранных товаров
- events: IEvents - экземпляр класса `EventEmitter`для инициации событий при изменении данных.

Так же класс предоставляет набор методов для взаимодействия с этими данными.

- setPayInfo(UserData: IUser): void - сохраняет данные покупателя в классе
- clearUser(): void - очищает данные о покупателе

### СЛОЙ ПРЕДСТАВЛЕНИЯ

Все классы представления отвечают за отображение внутри контейнера (DOM-элемент) передаваемых в них данных.

#### Класс Modal

Реализует модальное окно. Так же предоставляет методы `open` и `close` для управления отображением модального окна. Устанавливает слушатели на на клик в оверлей и кнопку-крестик для закрытия попапа.

- constructor(events: IEvents) Конструктор принимает экземпляр класса `EventEmitter` для возможности инициации событий.

Поля класса

- сontent: HTMLElement - контент модального окна
- events: IEvents - брокер событий
- templateName: string - значение атрибута name темплейта

Методы:

- getContent: HTMLElement - геттер для получения контента темплейта
- setValid(isValid: boolean): void - изменяет активность кнопки подтверждения
- getInputValues(): Record<string, string> - возвращает объект с данными из полей формы, где ключ - name инпута, значение - данные введенные пользователем
- showInputError (field: string): void - отображает текст ошибки под указанным полем ввода
- hideInputError (field: string): void - очищает текст ошибки под указанным полем ввода

#### Класс Card

Отвечает за отображение карточки, задавая в карточке данные названия, изображения, цены. Класс используется для отображения карточек на странице сайта. В конструктор класса передается DOM элемент темплейта, что позволяет при необходимости формировать карточки разных вариантов верстки. В классе устанавливаются слушатели на все интерактивные элементы, в результате взаимодействия с которыми покупателя генерируются соответствующие события.\
Поля класса содержат элементы разметки элементов карточки. Конструктор, кроме темплейта принимает экземпляр `EventEmitter` для инициации событий.\
Методы:

- setData(cardData: IProduct): void - заполняет атрибуты элементов карточки данными.
- isInCart(): boolean - метод возвращает наличие товара в корзине
- deleteCard(): void - метод для удаления разметки карточки
- render(): HTMLElement - метод возвращает полностью заполненную карточку с установленными слушателями
- getCard():id возвращает уникальный id карточки

#### Класс CardsContainer

Отвечает за отображение блока с карточками на главной странице. Предоставляет метод сеттер `container` для полного обновления содержимого. В конструктор принимает контейнер, в котором размещаются карточки.

#### Класс CartInfo

Отвечает за блок сайта с информацией о корзине. Принимает в конструктор контейнер - элемент разметки блока корзины и экземпляр `EventEmitter` для инициации событий при нажатии пользователем на иконку корзины. Устанавливает в конструкторе слушатели на иконку корзины.\

Поля класса:

- total: number - общая сумма заказа
- CartIcon: HTMLElement - ссылка на все разметку блока

Методы:

- setTotal(): TCartTotal - устанавливает данные об общей ценности корзины

#### Класс AppApi

## Взаимодействие компонентов

Код, описывающий взаимодействие представления и данных между собой находится в файле `index.ts`, выполняющем роль презентера.\
Взаимодействие осуществляется за счет событий генерируемых с помощью брокера событий и обработчиков этих событий, описанных в `index.ts`\
В `index.ts` сначала создаются экземпляры всех необходимых классов, а затем настраивается обработка событий.

Список всех событий, которые могут генерироваться в системе:\
События изменения данных (генерируются классами моделями данных)

- `user:changed` - изменение данных покупателя
- `cart:changed` - изменение корзины

События, возникающие при взаимодействии покупателя с интерфейсом

- `cart:open` - открытие модального окна с формой корзины
- `card:open` - открытие модального окна карточки
- `card:add` - выбор карточки для добавления её в корзину
- `card:delete` - выбор карточки для удаления её из корзины
- `edit-order:input` - изменение данных в форме с данными о способе оплаты
- `edit-order:submit` - сохранение данных в форме с данными о способе оплаты
- `edit-order:validation` - событие, сообщающее о необходимости валидации в форме с данными о способе оплаты
- `edit-contacts:input` - изменение данных в форме с данными о контактной информацией
- `edit-contacts:submit` - сохранение данных в форме с данными о контактной информацией
- `edit-contacts:validation` - событие, сообщающее о необходимости валидации в форме с данными о контактной информацией
- `remove-cart:submit` - событие, генерируемое при нажатии "Оплатить" форме с данными о контактной информацией
